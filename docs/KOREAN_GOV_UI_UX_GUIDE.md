# 韩国政府 UI/UX 设计指南

## 设计原则

本指南基于 **KRDS（디지털 정부서비스 UI/UX 가이드라인）** 官方设计系统，适用于江原特别自治道创业企业门户项目。

### 核心设计理念

1. **正式性与专业性**：强调正式、专业、可信赖的视觉形象
2. **可访问性优先**：优先考虑可读性和易用性，确保信息清晰传达
3. **保守的视觉风格**：采用较大的间距、清晰的层次结构，避免过于紧凑的布局
4. **8-Point Grid 系统**：所有间距和尺寸使用 8px 的倍数，特定情况下可使用 4px 的倍数进行微调
5. **响应式设计**：移动端使用较小间距，桌面端使用较大间距，适应不同设备

### 间距系统特点

- **基础单位**：8px（8-Point Grid 系统）
- **屏幕边距**：桌面端最小 24px，移动端最小 16px
- **响应式原则**：移动端间距较小，桌面端间距较大
- **垂直间距**：通常比水平间距大 1.25x - 1.5x

### 布局系统

- **列布局**：桌面端默认 12 列，可扩展至 16 列
- **响应式网格**：基于常见设备尺寸设置断点
- **内容区域**：设置最大宽度，确保布局一致性

## 间距规范

### 间距等级

间距系统分为以下等级（从紧凑到宽松）：
- **none**：无间距
- **tight**：紧密排列
- **small**：小间距
- **default**：标准间距（最常用）
- **large**：大间距
- **xlarge**：超大间距

### 间距类型

#### 1. Padding（内边距）
- **用途**：组件内部内容与边框的距离
- **应用场景**：容器、按钮、卡片、表单输入框等元素内部空间

#### 2. Margin（外边距）
- **用途**：独立组件之间的外部距离
- **应用场景**：区块之间、内容区域、单边留白

#### 3. Gap（布局间距）
- **用途**：Flexbox/Grid 布局容器中组件之间的间距
- **应用场景**：网格布局、Flex 容器中的统一间距

#### 4. Space（子元素间距）
- **用途**：组件内所有子元素的统一间距
- **应用场景**：表单字段、列表项、按钮组、导航项

### 间距选择原则

| 场景 | 使用方式 | 说明 |
|------|---------|------|
| 组件内部内容与边框的距离 | Padding | 元素内部空间 |
| Flexbox/Grid 布局中元素之间的间距 | Gap | 布局容器内的统一间距 |
| 组件之间的间距（非布局容器） | Margin | 独立元素之间的间距 |
| 父容器内所有子元素之间的统一间距 | Space | 自动为所有子元素添加间距 |

### 响应式设计

- 所有间距都应包含响应式变体（至少包含 `sm:` 和 `lg:`）
- 移动端优先设计（先写移动端类名，再添加响应式前缀）
- 移动端使用较小间距，桌面端使用较大间距

## 排版规范

### 字体系统

- **字体族**：系统字体栈，优先使用韩文字体（Noto Sans KR），支持中文字体（Noto Sans SC）
- **字体大小**：基于 8px 倍数，从 xs 到 6xl 的等级系统
- **行高**：正文 1.6，标题 1.3，确保可读性
- **字重**：正常（400）、中等（500）、半粗（600）、粗体（700）
- **字间距**：正常（0）为默认，标题可使用宽松字间距

### 标题层级

使用语义化的 HTML 标签（`<h1>` 到 `<h6>`），保持层级一致性：
- **H1**：页面主标题
- **H2**：页面副标题、区块标题
- **H3**：区块子标题
- **H4-H6**：卡片标题、列表标题等

### 文本颜色

- **主文本**：深色，用于正文和标题
- **次要文本**：中等灰色，用于辅助信息
- **链接文本**：主色调，用于可点击文本
- **强调文本**：强调色，用于警告、错误、重要提示

---

## 颜色系统规范

### 颜色分类

颜色系统分为以下类别：
- **主色调**：品牌主色，用于重要元素和强调
- **辅助色**：补充主色调，用于次要元素
- **语义色**：表示状态和含义（成功、警告、错误、信息）
- **中性色**：用于文本、背景、边框等基础元素

### 颜色使用原则

- **保守配色**：采用正式、专业的配色方案，避免过于鲜艳的颜色
- **对比度要求**：确保文本与背景有足够的对比度，保证可读性
- **语义一致性**：相同含义使用相同颜色（如错误统一使用红色系）
- **层次分明**：通过颜色深浅建立视觉层次

### 颜色变体

每种颜色应包含多个变体：
- **基础色**：标准颜色值
- **浅色变体**：用于背景、悬停状态
- **深色变体**：用于强调、激活状态
- **禁用状态**：降低饱和度和透明度

## 布局规范

### 容器宽度

- **small**：表单页面、详情页、博客文章
- **default**：标准内容页面、列表页（最常用）
- **large**：首页横幅、全屏展示

### 响应式断点

- **移动端**：< 640px
- **小屏幕**：640px+
- **中等屏幕**：768px+
- **大屏幕**：1024px+
- **超大屏幕**：1280px+

### 页面边距

- **最小边距**：桌面端 24px，移动端 16px
- **响应式**：移动端较小，桌面端较大
- **内容居中**：使用 `mx-auto` 使内容水平居中

---

## 组件设计规范

### 按钮组件

- **尺寸等级**：提供多种尺寸，最小点击区域符合可访问性要求
- **样式变体**：主要按钮、次要按钮、文本按钮、图标按钮
- **状态设计**：默认、悬停、激活、禁用、加载状态
- **使用原则**：主要操作使用主要按钮，次要操作使用次要按钮

### 表单组件

- **输入框**：提供标准、大尺寸变体，包含标签、占位符、帮助文本
- **选择器**：下拉选择、单选、多选，保持一致的视觉风格
- **验证反馈**：错误、成功、警告状态的视觉提示
- **布局原则**：标签与输入框对齐，错误信息清晰可见

### 卡片组件

- **容器设计**：清晰的边框或阴影，适当的内边距
- **内容组织**：标题、正文、操作按钮的层次结构
- **交互反馈**：悬停、点击状态的视觉变化
- **响应式**：移动端和桌面端的布局适配

### 导航组件

- **导航栏**：清晰的层级结构，当前页面高亮显示
- **面包屑**：显示当前位置，支持快速导航
- **分页**：清晰的页码和导航按钮
- **标签页**：内容分组，当前标签明确标识

### 反馈组件

- **提示消息**：成功、错误、警告、信息的统一样式
- **加载状态**：加载指示器、骨架屏
- **空状态**：无数据时的友好提示
- **模态框**：确认对话框、信息展示弹窗

### 组件使用原则

- **一致性**：相同功能的组件使用相同的样式和交互
- **可访问性**：所有交互元素符合可访问性标准
- **响应式**：组件在不同设备上都能正常使用
- **保守设计**：符合政府网站的正式、专业风格

---

## 交互设计规范

### 交互状态

所有可交互元素应包含以下状态：
- **默认状态**：元素的初始外观
- **悬停状态**：鼠标悬停时的视觉反馈
- **激活状态**：点击或选中时的状态
- **焦点状态**：键盘导航时的焦点指示
- **禁用状态**：不可用时的视觉表现
- **加载状态**：异步操作进行中的反馈

### 交互反馈原则

- **即时反馈**：用户操作后立即提供视觉反馈
- **明确指示**：状态变化清晰可见，避免模糊不清
- **过渡动画**：使用平滑的过渡效果，提升用户体验
- **保守动画**：避免过于夸张的动画效果，保持正式感

### 交互行为

- **点击区域**：确保点击区域足够大，符合可访问性要求
- **键盘导航**：所有交互元素支持键盘操作
- **焦点管理**：合理的焦点顺序和焦点指示
- **错误处理**：清晰的错误提示和恢复路径

### 加载与等待

- **加载指示**：长时间操作显示加载状态
- **进度反馈**：可量化的操作显示进度条
- **超时处理**：操作超时时的友好提示
- **骨架屏**：内容加载时使用骨架屏提升体验

### 交互一致性

- **统一模式**：相同类型的交互使用相同的模式
- **可预测性**：交互行为符合用户预期
- **错误预防**：通过设计减少用户犯错的可能性
- **操作确认**：重要操作提供确认机制

## 最佳实践

1. **优先使用 Space**：为多个子元素添加统一间距时，优先使用 `space-y-*` 或 `space-x-*`
2. **避免边距塌陷**：使用 Gap 或 Space 可以避免 Margin 的边距塌陷问题
3. **组合使用**：可以组合使用多种间距方式，但要保持逻辑清晰
4. **重置默认间距**：使用 `m-0`、`p-0` 重置浏览器默认间距
5. **明确区分**：区分组件内间距（Padding/Space）和组件间间距（Margin/Gap）
6. **首页使用大间距**：首页、主要内容区域应使用 large 等级间距
7. **布局容器优先使用 Gap**：Flexbox/Grid 布局中的元素间距应使用 `gap-*`
