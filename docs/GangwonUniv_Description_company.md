2. 프로그램

4. 원스톱 지원

## 메뉴 트리 구조

  * 서브메뉴 없음

  * 성과입력
* 원스톱 지원
  * 1:1문의
  * 자주 묻는 질문

강원 대학교 창업고도화 홈페이지

홈페이지 목적 : 강원도에 있는 기업 성과관리 시스템.

기업들이 홈페이지에 가입한 후, 관리자에서 인증이 된 경우에 한하여 로그인이 가능하고,

각 기업들이 성과를 연도 분기별로 기록하고 관리하는 사이트.

관리자는 게시판 등록하고, 기업 조회(나이스디앤비 홈페이지를 통해 기업정보 OpenAPI로 추출) 및 관리.

<사용 기술>

Frontend 홈페이지 2개, Backend API 연동 구현.

Frontend: React, JS6, Tailwind

Backend: Python (추후 LLM 창업톡 구현할 수도 있음)

DB : ProgreSQL

# 3. 기업 홈페이지 메인  (6 page)

- 메인화면 구성:

상단 대 매뉴와 왼쪽 사이드의 서브메뉴

센터 부분 구성 : 메인배너(1) 이미지(클릭시 URL있으면 해당페이지 이동)

공지사항(관리자에서 입력한 내용 최근 5개 추출)

보도자료(관리자에서 입력한 최근 1개 썸네일 이미지 추출, 클릭시 해당 게시판으로 이동)

관리자에서 입력한 메인 배너(2) (작은 사이즈) – 관리자에서 메인배너는 2개 입력((1),(2))

하단 푸터 구성 : 공통파일 템플릿 처리

이용약관, 개인정보처리 방침 클릭시 레이어팝업처리(해당 내용은 txt또는 html 가져와서 추출)

- 홈페이지 디자인 템플릿은 Tailwind 디자인 템플릿을 참조해서 작업한다.
- 대메뉴는 4개 : 시그템 소개, 프로그램 , 성과관리, 원스톱 지원

-각 대메뉴에는 각각 배너이미지가 자동으로 나온다.

- 관리자에서 대메뉴 별로 배너 입력, 총 5개 배너입력 필요-메인용, 각 대매뉴 4개 별 배너

## * 대메뉴 1 : 시스템 소개 (7 page)

-센터 부분의 배너는 관리자페이지의 배너관리에서 입력한 시스템소개 메뉴의 배너를 가져온다.

- url이 있으면 해당 페이지로 분기한다.
- 컨텐츠:

추후 입력 예정임. 관리자에서 콘텐츠 내용 수정할 수 있게 에디터 창으로 만들어 둔다.

즉, 관리자에서 각 페이지의 컨텐츠 관리 페이지가 필요.

## * 대메뉴2: 프로그램(8 page)

- 배너 이미지는 관리자 배너관리에서 입력된 프로그램 대메뉴의 배너를 추출한다. url있으면 클릭 추가
- 컨텐츠 :

공고 게시판 게시글 추출 :

화면에 한페이지에 10개의 개시글 추출 기본설정. – 게시글 개수는 선택가능하게(10개, 20개, 30개, 50개)

게시글 검색기능 추가 : 제목/내용 text로 검색가능하도록 구성

각 공고 게시글에는 “창업자 신청하기” 버튼을 둔다.

이 버튼 클릭 시 팝업 레이어 형태로 첨부파일 최대 5개까지 입력 가능하도록 하고 구성

저장할 데이터: 회원아이디,

첨부파일 최대 5개(이름은 폴더저장시 다른 이름으로 저장하고 디비에 원본 파일명 저장),

(첨부파일 폴더는 각 /upload/해당 기업 아이디/공고게시판/)

등록일자,

접수상태

각 공고 게시판 제목 클릭 시 상세 공고내용 페이지로 이동한다.

## * 대메뉴 3: 성과관리

### - 기업정보 (9 page)

- 좌측 서브메뉴 3개 : 기업정보, 성과조회, 성과입력
- 배너 이미지는 관리자 배너관리에서 입력된 프로그램 대메뉴의 배너를 추출한다. url있으면 클릭 추가
- 컨텐츠 : 기업정보 수정 페이지

기존 입력된 내용중 사업자번호(아이디)는 수정불가

나머지는 수정 가능하도록

### -성과조회 (10 page)

- 좌측 서브메뉴 3개 : 기업정보, 성과조회, 성과입력
- 배너 이미지는 관리자 배너관리에서 입력된 프로그램 대메뉴의 배너를 추출한다. url있으면 클릭 추가
- 컨텐츠 : 검색키워드 추가 : 년도별, 분기별, 상태값(select box:제출/보완/승인) 등의 상태별로 검색 가능하도록

하단에 성과입력 버튼 추가(서브 메뉴에도 성과입력 메뉴 있음)

성과 제출 후 관리자에서 승인한 후에만 이 페이지에서 확인 가능하다.

화면에 안내문구 추가 : “관리자 승인 전에는 관리자 승인후에만 화면에 보여진다”라는 안내 문구 필요.

해당 기업의 성과자료만 나오도록 구성한다.- 다른 기업의 성과관리가 나오면 안된다. (주의)

해당 기업이 직접 올린 성과관리는 삭제/수정이 가능해야 한다.

모든 첨부파일은 pdf만 가능.

승인된 게시글은 본인 기업의 성과관리 업로드 게시글 확인이 가능하고 파일을 다운로드 가능해야 한다.

게시글의 추출 항목 : 문서유형, 파일명, 문서상태, 문서확인(다운로드 버튼)

# [모든 페이지 공통사항]

* 모든 파일 저장의 형태는 upload폴더아래에 기업아이디 숫자로 폴더 생성 후 저장.
* 파일 저장 시 이름은 원본파일 이름 그대로 사용하지 말고, 변환된 이름을 이용하고 원본이름은 디비 테이블에 저장
* 첨부파일 수정 삭제 시 기존에 첨부된 파일은 정상 처리 후 삭제처리한다. 파일 이름은 유니크 해야 한다.
* 모든 데이터 입력 수정 삭제 처리시 transaction으로 묶고 roll back 처리.
* text입력항목은 SQL injection 방어 처리 필요
